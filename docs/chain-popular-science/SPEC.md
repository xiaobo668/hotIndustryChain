# 产业链科普口播 · 生成规范（SPEC）

> 版本：2026-06 · 适用：短视频口播 / 图文榜单 / 视频号 / 今日头条  
> 定位：产业知识科普，**合规无荐股**，可与榜单截图配合使用

---

## 1. 固定结构（五段式，不可打乱顺序）

| 序号 | 段落 | 时长建议 | 核心任务 |
|------|------|----------|----------|
| 0 | **开篇定调** | 3–5 秒 | 抓注意力 + 声明科普属性 + 合规兜底 |
| 1 | **上游** | 20–30 秒 | 基础原材料 / 核心元器件（产业链地基） |
| 2 | **中游** | 25–35 秒 | 代工 / 封装 / 零部件制造（加工厂） |
| 3 | **下游** | 20–30 秒 | 终端产品 / 落地场景（市场需求端） |
| 4 | **收尾升华** | 15–20 秒 | 需求传导 + 国产替代 + 合规结束语 |

**总时长建议：** 90–120 秒（短视频） / 800–1200 字（图文）

---

## 2. 各段写作规范

### 2.0 开篇定调

**必须包含：**
- 痛点：「很多朋友分不清 XX 产业链上下游…」
- 承诺：「用最通俗的方式完整拆解」
- 合规：「全程只做产业知识科普，不涉及个股投资建议」

**禁止：** 涨跌、估值、买卖、推荐、目标价

**模板：**
```
很多朋友分不清{产业链名称}上下游到底都是做什么的，今天抛开复杂的专业术语，用最通俗的方式，把整条产业链从源头原材料，到工厂生产，再到最终落地的终端产品，完整拆解一遍，全程只做产业知识科普，不涉及个股投资建议。
```

---

### 2.1 上游——基础原材料、核心元器件

**通俗逻辑：** 整条产业的「原材料仓库」，决定成本与性能上限。

**必须包含：**
1. 细分品类（2–4 个）
2. 通俗举例（生活化类比 + 该产业真实物料名）
3. 科普重点：壁垒高、技术门槛、国产替代进度

**模板要素：**
- `upstream.categories[]` — 品类列表
- `upstream.example` — 一句话举例
- `upstream.analogy` — 生活类比（如「芯片的印刷原料」）
- `upstream.highlight` — 科普重点一句

---

### 2.2 中游——代工、封装、零部件制造

**通俗逻辑：** 产业链的「生产车间」，产能与订单最集中。

**必须包含：**
1. 细分品类
2. 加工流程举例（原料→成品/模组）
3. 科普重点：扩产、产能、在手订单、景气度最先体现

**榜单配合：** 本段最适合插入 **产能榜 / 订单榜 Top10 截图**

---

### 2.3 下游——终端产品、落地应用场景

**通俗逻辑：** 最终买家，需求好坏决定全链扩产。

**必须包含：**
1. 细分场景（2–4 个）
2. 需求传导举例（下游采购→带动中上游）
3. 科普重点：需求拐点是景气周期起点

**榜单配合：** 可插入 **下游订单 / 应用场景相关榜单截图**

---

### 2.4 收尾升华

**固定三块（顺序不可变）：**

1. **需求传导逻辑**  
   下游需求↑ → 中游订单满、扩产 → 上游原材料采购↑；反之去库存。

2. **国产替代视角**  
   上中下游逐步实现自主供货，是长期成长逻辑（不讲个股）。

3. **合规兜底结束语**  
   ```
   以上仅为产业链行业科普梳理，所有数据来源于企业公开披露信息，不构成任何证券投资、交易操作建议。
   ```

## 4. 合规红线（生成后必检）

| 类型 | 禁止 | 替换为 |
|------|------|--------|
| 投资暗示 | 龙头、核心供应商、市占第一 | 规模靠前、主要供货企业 |
| 交易暗示 | 买入、卖出、抄底、逃顶 | （删除） |
| 预测暗示 | 预计涨停、目标价、必涨 | 产能、订单、需求、披露数据 |
| 荐股 | 关注XX股、布局XX | 产业链节点、环节、赛道 |

**允许使用的词：** 物料、产能、出货量、订单、需求、扩产、披露、年报、半年报、国产替代、景气度

---

## 5. 完播率加分技巧

1. **每段配一张榜单截图**：上游耗材 Top10 → 中游产能 Top10 → 下游订单 Top10  
2. **生活化类比**：晶圆厂 =「芯片印刷厂」、封测 =「芯片包装车间」、硅片 =「芯片的纸张」  
3. **数字只来自公开披露**：营收、产能、出货量需标注「据公开披露 / 年报 / 半年报」  
4. **口播节奏**：每段开头用「首先 / 接着 / 然后 / 最后」衔接  
5. **视觉字幕**：上游绿、中游蓝、下游橙、总结灰（可选）

---

## 6. 生成工作流

```
1. 选定产业链 → 从 data.js / 现有榜单提取上中下游节点
2. 按 Schema 填 upstream / midstream / downstream 字段
3. 运行 node scripts/verify-chain-pop-science.js 合规复验
4. 合并 fullScript → 复制口播 / 录制 / 配图榜单截图
5. 发布前人工复读一遍，确认无禁词
```
